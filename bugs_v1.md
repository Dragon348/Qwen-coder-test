# Отчёт об ошибках приложения Dialog Portal (bugs_v1.md)

## Дата анализа: 2024-04-19

## Критические ошибки (Linting Errors)

### 1. Ошибка в `src/components/DialogGraph.tsx` (строка 76)
**Тип:** `@typescript-eslint/no-unused-vars`  
**Описание:** Параметр `_event` определён, но не используется.  
**Код:**
```typescript
onResponseDragStart: (response: Response, _event: React.MouseEvent) => {
  setDraggedResponse({ response, sourceNodeId: node.id });
}
```
**Рекомендация:** Удалить неиспользуемый параметр `_event` или использовать его.

---

### 2. Ошибка в `src/components/DialogGraph.tsx` (строка 149)
**Тип:** `@typescript-eslint/no-explicit-any`  
**Описание:** Использование типа `any` запрещено правилами линтера.  
**Код:**
```typescript
<CustomNode data={n.data as any} selected={n.id === selectedNodeId} />
```
**Рекомендация:** Определить правильный тип для `data` вместо использования `any`. Например, создать интерфейс `CustomNodeData`.

---

### 3. Ошибка в `src/components/DialogGraph.tsx` (строка 220)
**Тип:** `react-refresh/only-export-components`  
**Описание:** Fast refresh работает только когда файл экспортирует только компоненты. Хук `useNodeDrag` экспортируется из файла с компонентами.  
**Код:**
```typescript
export const useNodeDrag = (...) => {...}
```
**Рекомендация:** Переместить хук `useNodeDrag` в отдельный файл (например, `hooks/useNodeDrag.ts`) или в существующий файл хуков.

---

### 4. Ошибка в `src/hooks/useDialogStore.ts` (строка 17)
**Тип:** `react-hooks/set-state-in-effect`  
**Описание:** Вызов `setState` синхронно внутри эффекта может вызвать каскадные перерисовки. Это нарушает рекомендации React.  
**Код:**
```typescript
useEffect(() => {
  const saved = localStorage.getItem(STORAGE_KEY);
  if (saved) {
    try {
      const data: DialogProject = JSON.parse(saved);
      setNodes(data.nodes || []);  // ← Ошибка здесь
      setProjectName(data.name || 'Новый проект');
    } catch (e) {
      console.error('Failed to load project from localStorage:', e);
    }
  }
}, []);
```
**Рекомендация:** Хотя данный случай технически безопасен (эффект выполняется только один раз при монтировании), формально это нарушение правила. Можно использовать паттерн с lazy initialization в `useState` или подавить предупреждение комментарием, если поведение корректно.

## Потенциальные проблемы архитектуры

### 5. Конфликт библиотек React Flow
**Файл:** `package.json`  
**Описание:** В зависимостях установлены обе библиотеки:
- `@xyflow/react`: ^12.10.2 (новая версия, ребрендинг React Flow)
- `reactflow`: ^11.11.4 (старая версия)

Это может привести к конфликтам и увеличению размера бандла.  
**Рекомендация:** Удалить `reactflow` и использовать только `@xyflow/react`.

---

### 6. Несогласованное использование координат при перетаскивании
**Файлы:** `App.tsx`, `DialogGraph.tsx`  
**Описание:** В `App.tsx` при перетаскивании узла координаты делятся на `scale`:
```typescript
updateNodePosition(draggedNodeId, {
  x: (event.clientX - dragOffset.x) / scale,
  y: (event.clientY - dragOffset.y) / scale
});
```
Но в `DialogGraph.tsx` логика определения целевого узла при drop не учитывает масштаб:
```typescript
const nodeX = node.position?.x || 0;
const nodeY = node.position?.y || 0;
return x >= nodeX && x <= nodeX + 280 && y >= nodeY && y <= nodeY + 200;
```
**Рекомендация:** Привести логику работы с координатами к единому стандарту с учётом масштаба.

---

### 7. Отсутствие обработки ошибки при создании соединения
**Файл:** `DialogGraph.tsx` (строки 114-116)  
**Описание:** При создании соединения через drag-and-drop нет проверки, что соединение не будет дубликатом.  
**Рекомендация:** Добавить проверку на существование уже созданного соединения перед его созданием.

---

### 8. Генерация ID может создавать коллизии
**Файл:** `types.ts` (строки 31-33)  
**Описание:** Функция `generateId()` использует `Math.random()`, что теоретически может привести к коллизиям при большом количестве узлов.  
**Код:**
```typescript
export const generateId = (): string => {
  return Math.random().toString(36).substring(2, 15) + Math.random().toString(36).substring(2, 15);
};
```
**Рекомендация:** Использовать более надёжный генератор UUID, например `crypto.randomUUID()` или библиотеку `uuid`.

---

## Проблемы UX/UI

### 9. Отсутствие подтверждения при удалении узла
**Файл:** `NodeEditor.tsx` (строка 40)  
**Описание:** Удаление узла происходит без подтверждения, что может привести к случайной потере данных.  
**Рекомендация:** Добавить модальное окно подтверждения перед удалением.

---

### 10. Нет визуальной индикации битых ссылок
**Файл:** `DialogGraph.tsx`  
**Описание:** При валидации обнаруживаются битые ссылки, но они не отображаются визуально на графе.  
**Рекомендация:** Добавить визуальную индикацию (красный цвет, иконка предупреждения) для соединений, ведущих к несуществующим узлам.
