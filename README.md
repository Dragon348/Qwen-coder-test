# Веб-портал создания цепочек диалогов

Веб-приложение для визуального проектирования нелинейных диалогов с системой требований к характеристикам персонажа.

## Описание

Портал позволяет создавать интерактивные диалоговые цепочки, где каждый узел (окошко) представляет собой реплику или событие, а переходы между узлами определяются выбором игрока.

## Основные понятия

### Окошко (Диалоговый узел)

Базовый элемент диалога, содержащий:

- **Заголовок** — краткое название или тема реплики
- **Основной текст** — содержание диалога, описание ситуации
- **Варианты ответов** — список доступных действий/реплик игрока

### Вариант ответа

Каждый ответ может иметь:

- **Текст ответа** — что говорит или делает игрок
- **Требование** (опционально) — условие для доступности этого варианта

#### Формат требований

Требования указываются в формате `<Характеристика: Значение>`:

```
<Сила: 10>
<Ловкость: 8>
<Интеллект: 12>
<Репутация: 5>
```

Ответ с требованием будет доступен для выбора только если соответствующая характеристика персонажа удовлетворяет условию.

## Пример структуры

```
┌─────────────────────────────────┐
│ Заголовок: Встреча у ворот     │
│                                 │
│ Текст: Стражник преграждает    │
│ вам путь и требует пропуск.    │
│                                 │
│ Ответы:                        │
│ • Показать пропуск             │
│ • <Сила: 10> Прорваться силой  │
│ • <Харизма: 8> Уговорить       │
└─────────────────────────────────┘
```

## Возможности

- 📝 Создание и редактирование диалоговых узлов
- 🔗 Визуализация связей между узлами с помощью кривых Безье (не пересекают блоки)
- ⚙️ Настройка условий доступа к вариантам ответов
- 💾 Сохранение и загрузка проектов
- 🎯 Валидация требований
- 🛡️ Подтверждение перед удалением узлов
- 📤 Экспорт JSON без служебных данных (позиции узлов не экспортируются)

## Использование

1. Создайте новый проект
2. Добавьте окошки с текстом диалога
3. Настройте варианты ответов
4. При необходимости добавьте требования к характеристикам
5. Свяжите ответы с последующими узлами (перетаскиванием ответа на целевой узел)
6. Сохраните и экспортируйте результат

## Установка и запуск

```bash
cd dialog-portal
npm install
npm run dev
```

## Технологии

- React 19
- TypeScript
- @xyflow/react (React Flow) — библиотека для визуализации графов
- Vite — сборщик проектов

## Примечания

- Позиции узлов сохраняются только локально в браузере (localStorage) и не экспортируются в JSON
- При импорте проекта узлы автоматически располагаются сеткой
- Для генерации уникальных ID используется `crypto.randomUUID()` (с фолбэком на `Math.random()`)
- Соединения между узлами рисуются кривыми Безье для избежания пересечений с блоками
