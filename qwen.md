# Инструкции для ИИ-агента по разработке портала диалоговых цепочек

## Цель
Разработать веб-портал для визуального проектирования нелинейных диалогов с системой требований к характеристикам персонажа.

## Основные требования

### 1. Архитектура приложения
- Создать одностраничное веб-приложение (SPA)
- Реализовать клиентскую часть с интерактивным визуальным редактором
- Обеспечить локальное сохранение данных (localStorage) или экспорт/импорт проектов

### 2. Структура данных

#### Диалоговый узел (Окошко)
Каждый узел должен содержать:
```typescript
interface DialogNode {
  id: string;              // Уникальный идентификатор
  title: string;           // Заголовок узла
  text: string;            // Основной текст диалога
  responses: Response[];   // Список вариантов ответов
}
```

#### Вариант ответа
```typescript
interface Response {
  id: string;                    // Уникальный идентификатор
  text: string;                  // Текст ответа игрока
  requirement?: Requirement;     // Опциональное требование
  nextNodeId?: string;           // Ссылка на следующий узел
}
```

#### Требование
```typescript
interface Requirement {
  characteristic: string;  // Название характеристики (Сила, Ловкость, Интеллект, Репутация и т.д.)
  value: number;           // Минимальное требуемое значение
}
```

### 3. Функциональные требования

#### 3.1 Создание и редактирование узлов
- Добавить возможность создания новых диалоговых узлов
- Реализовать редактирование заголовка и основного текста
- Предусмотреть удаление узлов

#### 3.2 Управление вариантами ответов
- Добавление/удаление вариантов ответов в узле
- Редактирование текста ответа
- Привязка ответа к целевому узлу (создание связей)
- Добавление опциональных требований к ответам

#### 3.3 Система требований
- Поддержка формата требований: `<Характеристика: Значение>`
- Примеры характеристик: Сила, Ловкость, Интеллект, Репутация, Харизма
- Валидация формата требований при вводе
- Визуальное отображение требований в интерфейсе

#### 3.4 Визуализация
- Отображение узлов в виде блоков/карточек
- Визуальное представление связей между узлами (линии/стрелки)
- Поддержка перетаскивания узлов (drag-and-drop)
- Масштабирование и навигация по полю диалога

#### 3.5 Сохранение и загрузка
- Сохранение проекта в локальном хранилище браузера
- Экспорт проекта в JSON-файл
- Импорт проекта из JSON-файла
- Авто-сохранение при изменениях

#### 3.6 Валидация
- Проверка наличия битых ссылок (ответы ведут к несуществующим узлам)
- Валидация формата требований
- Предупреждение о узлах без исходящих связей (тупики)

### 4. Технические требования

#### Фронтенд
- Использовать современный JavaScript/TypeScript
- Применить фреймворк: React, Vue.js или Svelte (на выбор разработчика)
- Для визуализации графов использовать библиотеку: 
  - React Flow (для React)
  - Vue Flow (для Vue)
  - Или аналогичную
- Адаптивный дизайн для работы на разных экранах

#### Стили
- Чистый и интуитивно понятный интерфейс
- Цветовое кодирование для различных элементов
- Поддержка тёмной/светлой темы (опционально)

#### Производительность
- Оптимизация рендеринга для больших диалоговых деревьев
- Ленивая загрузка элементов при необходимости

### 5. Этапы разработки

#### Этап 1: Базовая структура
1. Настроить проект с выбранным фреймворком
2. Определить типы данных и структуру состояния
3. Создать базовые компоненты для узлов и ответов

#### Этап 2: Редактирование
1. Реализовать создание/удаление узлов
2. Добавить формы редактирования содержимого узлов
3. Реализовать управление вариантами ответов

#### Этап 3: Визуализация графа
1. Интегрировать библиотеку для работы с графами
2. Реализовать отображение связей между узлами
3. Добавить drag-and-drop функциональность

#### Этап 4: Система требований
1. Реализовать парсинг формата требований
2. Добавить UI для добавления/редактирования требований
3. Внедрить валидацию требований

#### Этап 5: Сохранение и экспорт
1. Реализовать локальное сохранение
2. Добавить экспорт в JSON
3. Добавить импорт из JSON

#### Этап 6: Тестирование и полировка
1. Протестировать все сценарии использования
2. Исправить ошибки
3. Улучшить UX/UI

### 6. Критерии приёмки

- [ ] Можно создать новый проект с нуля
- [ ] Можно добавлять и редактировать диалоговые узлы
- [ ] Можно создавать варианты ответов с текстом
- [ ] Можно добавлять требования к ответам в формате `<Характеристика: Значение>`
- [ ] Можно связывать ответы с другими узлами
- [ ] Визуально отображаются все узлы и связи между ними
- [ ] Проект можно сохранить и загрузить
- [ ] Проект можно экспортировать и импортировать как JSON-файл
- [ ] Работает валидация требований и связей

### 7. Пример структуры проекта

```

│                                 │
│ Текст: Стражник преграждает    │

│                                 │

```

### 8. Рекомендации по реализации

- Начните с минимально жизнеспособного продукта (MVP)
- Постепенно добавляйте функции согласно этапам
- Пишите чистый, документированный код
- Используйте систему контроля версий (Git)
- Тестируйте функциональность после каждого этапа
